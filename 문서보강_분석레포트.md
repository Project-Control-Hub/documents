## 5. 업계 표준 대비 주요 보강점

### PMBOK / 프로젝트 관리

| 항목 | 현재 상태 | 보강 방향 |
|------|----------|----------|
| 버퍼 일정 (Contingency Reserve) | 없음 | 각 단계별 1~2주 버퍼 |
| 크리티컬 패스 분석 | 없음 | Gantt에서 지연 영향 경로 식별 |
| 변경 관리 절차 | 없음 | CR 접수→영향분석→승인→반영 |
| 산출물 목록 (WBS Dictionary) | 부분적 | 단계별 산출물 전체 목록 |
| 완료 기준 (Exit Criteria) | 없음 | 각 단계별 완료 기준 정의 |

### Agile/Scrum

| 항목 | 현재 상태 | 보강 방향 |
|------|----------|----------|
| User Story 형식 | 미적용 | "As a [역할], I want [기능], So that [가치]" |
| Acceptance Criteria | 3개 FR만 | Given-When-Then 형식 전체 적용 |
| Sprint 단위 일정 | 없음 | 2주x4 스프린트, Sprint Goal 사전 정의 |
| DoR/DoD | 계획서에 없음 | 품질 관리 섹션으로 포함 |

### DevSecOps

| 항목 | 현재 상태 | 보강 방향 |
|------|----------|----------|
| SAST/DAST | 없음 | SonarQube, OWASP ZAP 파이프라인 통합 |
| 컨테이너 스캔 | 없음 | Trivy 이미지 스캔 |
| 의존성 취약점 | 없음 | npm audit, OWASP Dependency-Check |
| 시크릿 관리 | 평문 환경 변수 | AWS Secrets Manager |
| Signed Commits | 없음 | GPG/SSH 서명 정책 검토 |

### 테스트

| 항목 | 현재 상태 | 보강 방향 |
|------|----------|----------|
| 테스트 피라미드 비율 | 미정의 | 단위:통합:E2E = 70:20:10 목표 |
| 계약 테스트 | 없음 | OpenAPI 스펙 기반 Contract Testing |
| 접근성 테스트 | 없음 | WCAG 2.1 AA 기준 |
| Chaos Engineering | 없음 | 장애 주입 테스트 (DB/Redis 장애) |
