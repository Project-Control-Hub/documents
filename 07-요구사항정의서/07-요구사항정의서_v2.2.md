### 1.2 요구사항 상세

#### FR-001: 이슈 CRUD

| 항목 | 내용 |
|------|------|
| ID | FR-001 |
| 분류 | 이슈 관리 |
| 우선순위 | 필수 |
| 설명 | 사용자는 이슈(Epic/Story/Task/Bug/Sub-task)를 생성, 조회, 수정, 삭제할 수 있다 |

**상세 조건**:
- [ ] 이슈 제목은 `[모듈] 기능 요약` 형식 권장
- [ ] Epic은 2주 이상 소요되는 큰 기능 단위
- [ ] Story는 1 Sprint 내 완료 가능한 사용자 기능 (SP ≤ 8 권장)
- [ ] Bug 등록 시 재현 절차, 기대 결과, 실제 결과, 환경 정보 필수
- [ ] Sub-task는 상위 Story/Task의 세부 병렬 작업

**관련 화면**: SCR-006 (이슈 상세), SCR-007 (이슈 생성)
**관련 API**: POST/GET/PUT/DELETE /rest/api/3/issue

#### FR-002: 이슈 타입별 생성 (Epic/Story/Task/Bug/Sub-task)

| 항목 | 내용 |
|------|------|
| ID | FR-002 |
| 분류 | 이슈 관리 |
| 우선순위 | 필수 |
| 설명 | 5가지 이슈 타입(Epic/Story/Task/Bug/Sub-task)을 각 타입의 생성 기준에 맞게 등록할 수 있다 |

**상세 조건**:
- [ ] Epic: 목표, 완료 기준, 연관 Story 목록을 필수 입력
- [ ] Story: As-a/I want/So that 형식의 사용자 관점 기능 1개, SP 산정 필수
- [ ] Task: 사용자 기능이 아닌 기술적 작업(환경 설정, 리팩토링 등), 작업 목표 및 예상 소요 시간 기재
- [ ] Bug: 재현 절차, 기대 결과, 실제 결과, 환경 정보(OS/브라우저/API 버전/재현율) 필수 입력
- [ ] Sub-task: 상위 Story 또는 Task와 연결 필수, 담당자 및 예상 소요 시간 기재
- [ ] 이슈 크기 적정성 검증: SP 13 이상 이슈에 분리 검토 경고 표시

**관련 화면**: SCR-007 (이슈 생성)
**관련 API**: POST /rest/api/3/issue

#### FR-003: 이슈 상태 관리 (6단계 워크플로우)

| 항목 | 내용 |
|------|------|
| ID | FR-003 |
| 분류 | 이슈 관리 |
| 우선순위 | 필수 |
| 설명 | Backlog → Selected for Sprint → In Progress → Code Review → QA → Done 의 6단계 표준 상태를 통해 이슈 진행 현황을 관리한다 |

**상세 조건**:
- [ ] 6단계 표준 상태 각각의 담당자 역할 정의 (Backlog: PO, Selected for Sprint: SM, In Progress: Developer, Code Review: Reviewer, QA: QA Engineer, Done: 팀 전체)
- [ ] 이슈 상태는 보드 상의 컬럼과 1:1 매핑
- [ ] 상태 전환 시 전환 조건 검증 후 이동 허용 (섹션 8.3 전환 규칙 적용)
- [ ] 상태 변경 이력은 Audit Log에 자동 기록
- [ ] Done 전환 시 DoD(Definition of Done) 체크리스트 전체 충족 여부 확인

**관련 화면**: SCR-004 (스크럼 보드)
**관련 API**: POST /rest/api/3/issue/{issueKey}/transitions

#### FR-004: 우선순위 설정 (5단계)

| 항목 | 내용 |
|------|------|
| ID | FR-004 |
| 분류 | 이슈 관리 |
| 우선순위 | 필수 |
| 설명 | 이슈에 Highest/High/Medium/Low/Lowest 5단계 우선순위를 설정하여 작업 순서를 관리한다 |

**상세 조건**:
- [ ] 우선순위는 Highest / High / Medium / Low / Lowest 5단계로 구성
- [ ] 이슈 생성 시 기본값은 Medium으로 설정
- [ ] 백로그 및 보드 화면에서 우선순위 기준 정렬 지원
- [ ] 우선순위 변경 시 Audit Log에 변경 전/후 값 기록
- [ ] JQL에서 priority 필드로 필터링 가능 (예: priority = Highest)

**관련 화면**: SCR-006 (이슈 상세), SCR-010 (백로그)
**관련 API**: PUT /rest/api/3/issue/{issueKey}
