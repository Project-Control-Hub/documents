## 1. 소개

### 1.1 문서 목적

이 매뉴얼은 **Project Control Hub(PCH)** 를 사용하는 최종 사용자가 이슈·보드·스프린트·검색 등 핵심 기능을 이해하고 운영할 수 있도록 안내한다.

### 1.2 대상 독자

| 대상 | 설명 |
|------|------|
| 일반 사용자 | 이슈 조회·생성, 보드에서 작업 이동, 댓글·알림 사용 |
| 애자일 역할 | Product Owner, Scrum Master, Developer, QA 등 |
| 관리자 | 프로젝트 설정, 권한, 워크플로 관련 설정(권한 범위 내) |

### 1.3 시스템 요구사항

| 항목 | 최소 사양 | 권장 사양 |
|------|----------|----------|
| 웹 브라우저 | Chrome 120+, Firefox 121+, Safari 17+ | 최신 버전 |
| 해상도 | 1280 × 720 | 1920 × 1080 이상 |
| 네트워크 | 안정적인 인터넷 | 유선 또는 10Mbps+ |
| 모바일 앱 | iOS 15+ / Android 12+ | 최신 OS, PCH Flutter 앱 |

## 2. 시작하기

### 2.1 계정 및 로그인

1. 조직에서 안내한 **PCH URL** 에 접속한다.
2. **로그인**에서 이메일(또는 SSO)으로 인증한다.
3. 초기 비밀번호 변경·2FA가 켜져 있다면 안내에 따른다.

| 로그인 방식 | 설명 |
|------------|------|
| 이메일/비밀번호 | 기본 로컬 계정 |
| SSO | 조직 OIDC/SAML 연동 시(도입 시 매뉴얼 갱신) |

### 2.2 초기 설정

1. **프로필**: 표시 이름, 아바타, 타임존.
2. **알림**: 이메일·인앱 알림 수준.
3. **모양**: 라이트/다크 모드(지원 시).

## 3. 주요 기능

### 3.1 대시보드

**설명**: 할당 이슈, 스프린트 진행, 간단한 차트를 한 화면에서 본다.

**사용 방법**:

1. 상단 메뉴에서 **대시보드**를 연다.
2. **가젯**(필터 결과, 번다운 등)을 역할에 맞게 배치한다.
3. 특정 이슈를 클릭하면 이슈 상세로 이동한다.

### 3.2 보드(스크럼 / 칸반)

**설명**: 스프린트 또는 지속 흐름에 따라 이슈를 컬럼(상태)별로 옮긴다.

**사용 방법**:

1. **프로젝트** → **보드**를 연다.
2. 카드를 드래그하여 상태를 변경한다(WIP 제한 초과 시 경고가 날 수 있음).
3. 칸반에서 **WIP** 상한은 프로젝트 설정을 따른다.

**팁**: 동일 이슈에 대한 동시 편집 시 최신 저장 기준으로 갱신될 수 있으므로, 중요한 필드 변경 후 새로고침으로 확인한다.

### 3.3 백로그 및 스프린트

**설명**: 우선순위 정렬된 이슈 목록에서 스프린트에 이슈를 넣거나 뺀다.

**사용 방법**:

1. **백로그** 화면에서 이슈 순서를 조정한다.
2. **스프린트 계획** 시 이슈를 스프린트로 끌어 올린다.
3. 스프린트 **시작/완료**는 Scrum Master 등 권한 보유자가 수행한다.

### 3.4 이슈 상세

**설명**: 설명, 첨부, 댓글, 링크된 이슈, 히스토리, Planning Poker(모달) 등을 다룬다.

**사용 방법**:

1. 목록 또는 보드에서 이슈 키(예: `PROJ-123`)를 연다.
2. **댓글**에 `@이름`으로 멘션할 수 있다.
3. **워치**로 변경 알림을 구독한다.

### 3.5 JQL 검색

**설명**: 구조화된 검색어로 이슈를 찾는다.

**예시**:

```text
project = PROJ AND status = "In Progress" AND assignee = currentUser()
ORDER BY priority DESC
```

### 3.6 로드맵(Epic)

**설명**: Epic 단위로 기간·의존 관계를 시각화한다(프로젝트에서 활성화된 경우).

## 5. 자주 묻는 질문 (FAQ)

### Q1. 카드를 옮겼는데 상태가 바로 먹지 않아요.

**A**: 네트워크 지연이나 권한 제한, 또는 **전환 조건**(예: 필수 필드 미입력)일 수 있습니다. 오류 토스트 메시지를 확인하고 이슈 상세에서 필수값을 채운 뒤 다시 시도하세요.

### Q2. JQL에서 `currentUser()`가 안 먹어요.

**A**: 지원 함수 목록은 PCH JQL 엔진 버전에 따릅니다. [요구사항 FR-016](../07-요구사항정의서/07-요구사항정의서_v2.2.md) 및 릴리즈 노트를 확인하세요.

### Q3. 모바일에서 보드가 다르게 보여요.

**A**: [반응형/접근성 가이드](../반응형_접근성_가이드.md)의 브레이크포인트에 따라 컬럼·터치 타겟이 조정됩니다.

## 6. 문제 해결

| 증상 | 가능 원인 | 해결 방법 |
|------|----------|----------|
| 로그인 후 빈 화면 | 캐시·확장 프로그램 충돌 | 시크릿 창, 캐시 삭제 |
| 실시간 알림이 안 옴 | SSE/WebSocket 차단 | 방화벽·프록시에서 허용 여부 확인 |
| 첨부 업로드 실패 | 용량·포맷 제한 | 정책 안내 확인, 압축 후 재시도 |
