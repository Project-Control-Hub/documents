## 6. 보강 우선순위 로드맵 (Phase)

### Phase 1 — 필수 (릴리즈 직전 게이트)

1. `13-테스트보고서`: 스프린트/릴리즈별 **실측**으로 §3~§8 채움; `10`의 게이트 기준과 명시적 대조.  
2. `doc crossref` (스킬 §8): **FR ↔ 화면 SCR ↔ API 그룹 ↔ TC ID** 표를 07 또는 별도 RTM에 반영.  
3. 구버전(`*_v1.0` 등)과 최신본 **용어·SCR** 불일치가 없는지 샘플링.

### Phase 2 — 중요 (운영·온보딩)

1. `15-운영매뉴얼`: **비상 연락망**, **CloudWatch/Grafana URL**, **백업 스냅샷 검증 주기** 기입.  
2. `14-사용자매뉴얼`: 스크린샷·조직별 로그인 URL·권한별 메뉴 노출.  
3. 스킬 **12=테스트보고서 / 13=배포** 와 저장소 번호 **역전**을 `README` 상단 안전 공지 또는 `Project_Control_Hub.md`에 **한 줄 경고**로 추가(선택).

### Phase 3 — 고도화

1. 자동화: 링크 깨짐 검사, Mermaid 렌더 검증, 버전 네이밍 린트.  
2. `doc analyze` 주기 운영(분기 1회) 및 본 레포트 버전 누적.
