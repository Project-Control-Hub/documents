# 반응형/접근성 가이드

## 1. 반응형 디자인

### 1.1 브레이크포인트

| 이름 | 너비 | 대상 |
|------|------|------|
| Desktop Large | >= 1440px | 대형 모니터 |
| Desktop | 1024px ~ 1439px | 일반 데스크톱 |
| Tablet | 768px ~ 1023px | 태블릿 |
| Mobile | < 768px | 모바일 |

### 1.3 보드 모바일 대체 인터랙션
- 드래그 앤 드롭 → 상태 변경 버튼 (터치 친화)
- 컬럼 탭 전환으로 상태별 이슈 확인
- 스와이프로 이슈 카드 Quick Action

## 2. 접근성 (WCAG 2.1 AA)

### 2.1 기준

| 원칙 | 기준 | 적용 |
|------|------|------|
| 인식 가능 | 1.1.1 텍스트 대안 | 모든 아이콘에 aria-label |
| 인식 가능 | 1.4.3 색상 대비 | 최소 4.5:1 (일반), 3:1 (대형) |
| 인식 가능 | 1.4.11 비텍스트 대비 | UI 컴포넌트 3:1 이상 |
| 조작 가능 | 2.1.1 키보드 | 모든 기능 키보드로 조작 가능 |
| 조작 가능 | 2.4.7 포커스 표시 | 포커스 아웃라인 2px solid |
| 이해 가능 | 3.3.1 에러 식별 | 에러 필드 빨간 테두리 + 텍스트 설명 |
| 견고한 | 4.1.2 이름/역할/값 | ARIA 역할, 상태 속성 |

### 2.2 키보드 단축키

| 단축키 | 동작 | 화면 |
|--------|------|------|
| C | 이슈 생성 | 전역 |
| / | 검색 포커스 | 전역 |
| J/K | 이슈 목록 상/하 이동 | 백로그, 검색 결과 |
| Enter | 이슈 상세 열기 | 이슈 선택 상태 |
| Esc | 모달/패널 닫기 | 전역 |
| T | 상태 전환 메뉴 | 이슈 상세 |
| A | 담당자 변경 | 이슈 상세 |

### 2.3 보드 키보드 네비게이션
- Tab: 컬럼 간 이동
- Arrow Keys: 컬럼 내 카드 이동
- Space: 카드 선택 (드래그 모드)
- Arrow Left/Right: 선택된 카드를 다른 컬럼으로 이동
- Enter: 이동 확정
- Esc: 이동 취소

### 2.4 스크린 리더 지원
| 요소 | ARIA 속성 | 예시 |
|------|-----------|------|
| 보드 컬럼 | role="list", aria-label="In Progress (3/5)" | WIP 현황 포함 |
| 이슈 카드 | role="listitem", aria-label="PROJ-142 Story High SP:3" | 핵심 정보 읽기 |
| 차트 | role="img", aria-label="번다운 차트: 잔여 21 SP" | 데이터 요약 |
| 드래그 | aria-grabbed, aria-dropeffect | 상태 안내 |
| 알림 | role="alert", aria-live="polite" | 실시간 알림 |
