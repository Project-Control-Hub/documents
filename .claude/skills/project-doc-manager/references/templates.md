## 2. ERD 다이어그램

```mermaid
erDiagram
    USER {
        bigint id PK "사용자 ID"
        varchar email UK "이메일"
        varchar password "비밀번호 (해시)"
        varchar name "이름"
        enum role "역할 (ADMIN, USER)"
        timestamp created_at "생성일"

    }

    %% 여기에 엔티티와 관계를 추가하세요

# {프로젝트명} API 정의서

## 1. API 개요

| 항목 | 내용 |
|------|------|
| Base URL | `https://api.{도메인}/v1` |
| 인증 방식 | Bearer Token (JWT) |
| Content-Type | application/json |
| 문자 인코딩 | UTF-8 |

## 2. 공통 사항

### 2.1 공통 응답 포맷

**성공 응답**:
```json
{
  "success": true,
  "data": { },
  "message": "요청이 성공적으로 처리되었습니다."
}
```

**에러 응답**:
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "에러 메시지"
  }
}
```

### 2.2 공통 에러 코드

| 코드 | HTTP Status | 설명 |
|------|-------------|------|
| UNAUTHORIZED | 401 | 인증 실패 |
| FORBIDDEN | 403 | 권한 없음 |
| NOT_FOUND | 404 | 리소스 없음 |
| VALIDATION_ERROR | 422 | 입력값 검증 실패 |
| INTERNAL_ERROR | 500 | 서버 내부 오류 |

### 2.3 페이지네이션

```json
{
  "data": [],
  "pagination": {
    "page": 1,
    "size": 20,
    "totalElements": 100,
    "totalPages": 5
  }
}
```

## 3. API 엔드포인트

### 3.1 인증 (Auth)

#### POST /auth/login - 로그인

| 항목 | 내용 |
|------|------|
| Method | POST |
| URL | /auth/login |
| 인증 | 불필요 |

**Request Body**:
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| email | string | Y | 이메일 |
| password | string | Y | 비밀번호 |

**Response (200)**:
| 필드 | 타입 | 설명 |
|------|------|------|
| accessToken | string | 액세스 토큰 |
| refreshToken | string | 리프레시 토큰 |
| expiresIn | number | 만료 시간 (초) |

## 1. 전체 화면 흐름도

```mermaid
flowchart LR
    LOGIN[로그인] --> MAIN[메인 화면]
    MAIN --> FEATURE_A[기능 A]
    MAIN --> FEATURE_B[기능 B]
    MAIN --> SETTINGS[설정]
    FEATURE_A --> DETAIL_A[상세 A]
    FEATURE_B --> DETAIL_B[상세 B]
    SETTINGS --> PROFILE[프로필]
    SETTINGS --> LOGOUT[로그아웃]
    LOGOUT --> LOGIN
```

## 2. 주요 시퀀스 다이어그램

### 2.1 로그인 흐름

```mermaid
sequenceDiagram
    actor U as 사용자
    participant V as 화면
    participant A as API
    participant D as DB

    U->>V: 로그인 페이지 접근
    V-->>U: 로그인 폼 표시
    U->>V: 이메일/비밀번호 입력
    V->>A: POST /auth/login
    A->>D: 사용자 인증 확인
    alt 인증 성공
        D-->>A: 사용자 정보
        A-->>V: JWT 토큰
        V-->>U: 메인 화면으로 이동
    else 인증 실패
        D-->>A: 실패
        A-->>V: 에러 응답
        V-->>U: 에러 메시지 표시
    end
```
